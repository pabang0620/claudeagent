# Verification Command

현재 코드베이스 상태에 대한 종합적인 검증을 실행합니다.

## 지침

이 정확한 순서로 검증 실행:

1. **빌드 확인**
   - 이 프로젝트의 빌드 명령 실행
   - 실패 시, 에러 보고 후 중단

2. **타입 확인**
   - TypeScript/타입 체커 실행
   - 모든 에러를 파일:줄로 보고

3. **린트 확인**
   - 린터 실행
   - 경고 및 에러 보고

4. **테스트 스위트**
   - 모든 테스트 실행
   - 통과/실패 개수 보고
   - 커버리지 백분율 보고

5. **Console.log 감사**
   - 소스 파일에서 console.log 검색
   - 위치 보고

6. **Git 상태**
   - 커밋되지 않은 변경사항 표시
   - 마지막 커밋 이후 수정된 파일 표시

## 출력

간결한 검증 보고서 생성:

```
검증: [PASS/FAIL]

빌드:    [OK/FAIL]
타입:    [OK/X errors]
린트:    [OK/X issues]
테스트:  [X/Y passed, Z% coverage]
시크릿:  [OK/X found]
로그:    [OK/X console.logs]

PR 준비: [YES/NO]
```

중요한 이슈가 있는 경우, 수정 제안과 함께 목록화.

## 인자

$ARGUMENTS는 다음이 가능:
- `quick` - 빌드 + 타입만
- `full` - 모든 확인 (기본값)
- `pre-commit` - 커밋과 관련된 확인
- `pre-pr` - 전체 확인 + 보안 스캔
