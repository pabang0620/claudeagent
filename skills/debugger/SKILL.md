# Debugger Skill

디버깅 및 기능 검증 전문 스킬. 사용자의 의도대로 기능이 동작하는지 확인하고, 문제가 있으면 원인을 파악하여 수정합니다.

## 핵심 원칙

### 1. 사용자 의도 확인 (최우선)
- **디버깅 전 반드시 사용자에게 의도한 기능을 질문**
- "이 기능이 어떻게 동작해야 하나요?"
- "어떤 결과를 기대하셨나요?"
- 의도를 명확히 이해한 후에만 디버깅 시작

### 2. 수동 테스트 (필수)
- **자동화 스크립트(py 등) 사용 금지**
- **전체 검색으로 대충 확인하지 않음**
- 하나하나 직접 실행하며 테스트
- 실제 사용자 시나리오대로 검증

### 3. 코드 정밀 분석
- 관련 코드를 꼼꼼히 읽고 이해
- 데이터 흐름 추적
- 상태 변화 파악
- 비동기 처리 확인

## 디버깅 프로세스

### Step 1: 의도 파악
```markdown
## 기능 확인 질문

안녕하세요! 디버깅을 시작하기 전에 몇 가지 확인이 필요합니다.

1. **기대 동작**: 이 기능이 어떻게 동작해야 하나요?
2. **현재 문제**: 지금 어떤 문제가 발생하나요?
3. **재현 방법**: 문제를 재현하는 방법이 있나요?
```

### Step 2: 코드 분석
```
1. 관련 파일 식별
2. 코드 흐름 파악
3. 데이터 흐름 추적
4. 의심 지점 표시
```

### Step 3: 수동 테스트
```
1. 개발 서버 실행
2. 기능 직접 실행
3. 콘솔/네트워크 확인
4. 각 단계별 결과 기록
```

### Step 4: 원인 분석
```
1. 의도 vs 실제 동작 비교
2. 어디서 차이가 발생하는지 파악
3. 근본 원인 식별
4. 수정 방안 도출
```

### Step 5: 수정 및 검증
```
1. 최소한의 수정으로 해결
2. 수정 후 다시 수동 테스트
3. 의도대로 동작하는지 확인
4. 부작용 없는지 확인
```

## 테스트 방법

### 프론트엔드 테스트
```bash
# 개발 서버 실행
cd front-end && npm run dev

# 브라우저에서 직접 확인
# - 해당 페이지 접속
# - 기능 직접 실행
# - 콘솔 에러 확인
# - 네트워크 탭 확인
```

### 백엔드 테스트
```bash
# 서버 실행
cd back-end && npm run dev

# API 직접 호출
curl -X POST http://localhost:3000/api/endpoint \
  -H "Content-Type: application/json" \
  -d '{"key": "value"}'

# 응답 확인
# - success 값 확인
# - data 내용 확인
# - msg 메시지 확인
```

### 통합 테스트
```
1. 백엔드 서버 실행
2. 프론트엔드 서버 실행
3. 실제 사용 시나리오대로 테스트
4. 전체 흐름이 의도대로 동작하는지 확인
```

## 문제 유형별 접근

### 기능 미작동
1. 코드가 실행되는지 확인 (console.log)
2. 조건문 확인
3. 이벤트 바인딩 확인
4. API 호출 확인

### 예상과 다른 결과
1. 입력 데이터 확인
2. 중간 처리 과정 추적
3. 상태 변화 확인
4. 출력 데이터 확인

### 에러 발생
1. 에러 메시지 정확히 읽기
2. 스택 트레이스 추적
3. 발생 시점 파악
4. 관련 코드 분석

### 간헐적 문제
1. 재현 조건 파악
2. 타이밍 문제 확인
3. 비동기 처리 확인
4. 상태 경쟁 확인

## 출력 형식

### 디버깅 시작 시
```markdown
# 디버깅 시작

## 의도 확인
{사용자에게 질문}
```

### 분석 결과
```markdown
# 디버깅 보고서

## 사용자 의도
{사용자가 원하는 동작}

## 현재 동작
{실제로 일어나는 동작}

## 문제 원인
- 파일: `{파일 경로}:{라인}`
- 원인: {상세 설명}

## 테스트 결과
1. {테스트 1}: ✅/❌
2. {테스트 2}: ✅/❌

## 수정 내용
```diff
- 기존 코드
+ 수정된 코드
```

## 검증 결과
- 의도대로 동작: ✅
- 부작용 없음: ✅
```

## 주의사항

### Do
- 사용자 의도 먼저 파악
- 하나씩 직접 테스트
- 코드를 꼼꼼히 읽기
- 최소한의 수정
- 수정 후 반드시 재검증

### Don't
- 의도 모른 채 디버깅 시작
- 자동화 스크립트로 테스트
- 전체 검색으로 대충 확인
- 여러 곳 동시에 수정
- 검증 없이 완료 선언
